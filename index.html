Core Capabilities

EVERY TOOL
A DBA NEEDS

From raw wait stats to plain-English business impact — OroDb connects all the signals in one place.

sys.dm_exec_query_stats

QUERY FINGERPRINTING

Normalizes parameterized SQL into stable identities. Two queries doing the same logical work share one fingerprint regardless of literal values — enabling true trend tracking over time.

Regression Detection
sys.dm_os_wait_stats

WAIT STATS ENGINE

Real-time and snapshot-based wait type analysis. Categorised into CPU, IO, Memory, Lock, and Other. Signal-vs-resource breakdown instantly tells you whether you have CPU pressure or data pressure.

Live Dashboard
sys.dm_exec_query_plan

PLAN REGRESSION DETECTION

Hashes the structural shape of execution plans across snapshots. When a query's plan changes and performance drops simultaneously, OroDb flags it immediately — with the old and new plan side by side.

Plan Shape Hash
sys.dm_db_missing_index_details

INDEX RECOMMENDATIONS

Classified as HighConfidence, ReviewRequired, ConflictingDuplicate, or WriteHeavyRisk — not just a raw DMV dump. Approval workflow tracks status through Pending → Approved → Applied with full audit trail.

Workflow + Classification
./incident_bundle.zip

INCIDENT BUNDLES

One click generates a complete ZIP: wait stats CSV, top queries TXT, execution plan HTML viewers, index recommendation scripts, alert history, sys.configurations snapshot, and a plain-English narrative summary ready for vendor escalation.

Export Ready
alert_rules.evaluate()

CONFIGURABLE ALERTS

Define thresholds per server: signal wait percentage, IO wait share, specific wait types. Evaluated on every snapshot with a 30-minute deduplication window. Critical alerts fire email notifications automatically.

CAPTURE.

ACT.

OroDb installs in minutes. Point it at any SQL Server instance and it begins building a performance history immediately — no agents, no schema changes on your monitored databases.

01

CONNECT YOUR SERVER

Add a monitored server with a read-only DMV login. Connection string encrypted at rest with AES-256 or stored in Azure Key Vault.

02

CAPTURE SNAPSHOTS

Manual on-demand or scheduled via cron. Each snapshot captures wait stats, top queries, and builds fingerprint trend points.

03

WATCH THE TIMELINE

All alerts, regressions, plan changes, and new queries correlated chronologically. See exactly what changed and when.

04

RESOLVE & EXPORT

Approve index recommendations, assign work to team members, export incident bundles, and track the baseline recovering over time.
